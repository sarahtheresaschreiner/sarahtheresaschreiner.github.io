    Der Switch

    • Vermittlungsstelle innerhalb eines Computernetzwerks
    • intelligenter Verteiler, der genau weiß, wie etwas weitergeleitet wird (wählt zwischen den möglichen kabeln aus) -> lenkt
    • Schlauere version als hub, siehe später, switch wählt hub nicht

    Datenfluss

    • ließt die MAC-Adresse des empfängers aus dem paket heraus
    • prüft seine switch-Tabelle (manchmal CAM-Tabelle), die ihm sagt an welchem port der empfänger hängt
    • wenn keine adresse bekannt: an alle ports, außer dem sender.
    • lernphase (ports aufnehmen in seine tabelle)

    einsatzgebiet

    • nahezu jedes netzwerk
    • büro und unternehmens netzwerk: computer, drucker, server verbunden für effiziente kommunikation und datenverwaltung -> vgl. Schule
    • rechenzentren, heimnetzwerk

    Der Router

    • vermittlung von datenpaketen zwischen verschidenen Netzwerken
    • analysiert die informationen der datenpakete und entscheidet auf welchem weg sie am besten ans ziel kommen

    Rolle

    • rückgrat des internets: informationen ihren bestimmungsort zuführen, zB webseite, email

    Wegfindung: Routingtabellen und Protokolle - verbinden mehrere kleine Netzwerke zu einem großen Netzwerk, ermöglicht den austausch von Daten zwischen unterschiedliche Netzwerksegmenten - Datensicherheit: oft features wie firewall, schützt vor unerwünschten zugiffen

    Funktionsweise: Statisches vs. Dynamisches Routing

    1. routen manuell konfigurieren und sind fest
      • Verbindung explizit definiert
      • hohe kontrolle
      • weniger flexibel, manuelle anpassung (wenn sich irgendwas ändert)
    2. automatische anpassung an änderungen:
      • Routing-Protokolle tauschen sie informationen aus und bestimmen den weg selbstständig
      • flexibler und skaliert besser mit größeren Netzwerken
      • zB Internet: riesiges netzwerk, änderung

    Modem (Modulator Demodulator)

    • tauscht signale aus: zwei entgeräte über unterschiedliche Übertragungswege (Telefon, Kabel, Glasfaser)
    • verwendet um die Kommunikation zwischen dem Router im Heimnetzwerk und der Gegenstelle im Internetprovider zu ermöglichen
    • modem für internetverbindung (evtl auch zu anderem gerät)
    • modem immer über internet, router können auch geräte, oft inzwischen modem integriert

    (De)Modulation

    • Digitale signale werden vom sendenden Modem auf eine Trägerfrequenz aufmoduliert und dann vom empfangenden Modem wieder demoduliert (verständliche Form)
    • Internet: Hochfrequentzmodulation
    • Router kann das nicht modelieren deswegen muss ein Modem zwischengeschaltet werden
    • Trägerfrequenz: kann man sich wie eine CD vorstellen, ist der Träger für digitale signale, wie tonträger

    Client (über englisch client aus lateinisch cliens wörtlich für „Klient“ oder „Kunde“)

    • "programm das die dienste eines servers in anspruch nimmt"
    • bezeichnet im Client-Server-Modell ein Computerprogramm, das auf dem Endgerät eines Netzwerks ausgeführt wird und mit einem Server kommuniziert. Man nennt auch ein Endgerät selbst, das Dienste von einem Server abruft, Client. quelle

    • clients fordern dienste von einem server an und erhalten diese Beispiel: Client(Kunde) bestellt beim Kommunikationsweg (Kellner) einen Service des Servers (Küche) quelle mit tollem bild

    Hub

    Switch: Schlauere version als hub, switch wählt hub nicht Er broadcastet. Dadurch sind nicht nur alle Ports belegt, sondern auch alle Hosts. Sie bekommen alle Datenpakete zugeschickt, auch wenn sie nicht die Empfänger sind. Für die Hosts bedeutet das auch, dass sie nur dann senden können, wenn der Hub gerade keine Datenpakete sendet. Sonst kommt es zu Kollisionen. quelle

    Host (zu Deutsch: Wirt, Gastgeber)

    Ein Hardware server (host) ist ein computer auf dem ein oder mehrere server (software laufen) - Ein Gerät(Hardware), das ressourcen oder dienste für andere geräte(clients) in einem netzwerk bereitstellt. - ZB email, datein oder andere services für andere stationen anbietet quelle
