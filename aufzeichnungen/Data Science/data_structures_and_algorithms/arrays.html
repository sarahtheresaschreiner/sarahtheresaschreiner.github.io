    Arrays

    04.11.2025

    An array is a data structure used to store multiple elements of the same data type in a single variable.
    Arrays allow indexed access to each element and provide an efficient way to organize and manipulate collections of data.
    The size of a basic (static) array is fixed once it is created.

    Static Arrays

    • The simplest form of arrays — fixed in size
    • Elements are stored contiguously in memory
    • Each element can be accessed using an index (starting at 0)

    Example

    arr = [10, 20, 30, 40, 50]
print(arr[2])     # Output: 30

    Characteristics

    • Fast random access → O(1)
    • Insertion or deletion can be costly because elements must be shifted
    • Memory allocated contiguously, which makes it efficient but inflexible in size

    Dynamic Arrays

    A Dynamic Array (e.g., ArrayList in Java or list in Python) automatically resizes when elements are added or removed. When the internal storage is full, a new larger array is created, and all elements are copied into it.

    Memory:

    • Internally store a reference to a static array
    • When capacity is exceeded:
    • A new array (typically 2× the old size) is created
    • All elements are copied to the new array
    • The old array is freed from memory

    Example (Python List)

    arr = [1, 2, 3]
arr.append(4)       # adds element dynamically
arr.remove(2)       # removes element
print(arr)          # Output: [1, 3, 4]

    Example (Java ArrayList)

    import java.util.ArrayList;

ArrayList<Integer> arr = new ArrayList<>();
arr.add(10);
arr.add(20);
arr.remove(0);
System.out.println(arr);   // Output: [20]

    Characteristics

    • Resizable — grows automatically when more space is needed
    • Slightly slower insertions when resizing occurs (requires copying)
    • Internally uses a static array as its storage

    Memory Structure (Aufbau im Speicher)

    Static Arrays

    • Stored as a contiguous block of memory
    • Each element’s address can be calculated as:
    Address(i) = adress_of_first + i * size_of_element

    Visualization (ASCII)

    Static Array (contiguous memory):
[10][20][30][40][50]
 ↑   ↑   ↑   ↑   ↑
 |   |   |   |   |
 +---+---+---+---+--> continuous memory block

Dynamic Array (ArrayList internal):
ArrayList → [ref] → [10][20][30][--empty--]
When full → new larger array → [10][20][30][40][50][--empty--][--empty--]

    Complexity

    Operation Static Array Dynamic Array
    Access O(1) O(1)
    Insert at end O(n) Amortized O(1)
    Insert at index O(n) O(n)
    Delete O(n) O(n)
    Resize – O(n) (copying elements)

    Arrays vs Linked Lists

    Feature Arrays Linked Lists
    Memory layout Contiguous Scattered (each node has a pointer)
    Access time O(1) (index) O(n) (sequential traversal)
    Insertion/Deletion O(n) O(1) (if node reference known)
    Memory overhead Low Higher (extra pointer per node)
    Cache efficiency High Low

    Summary

    • Arrays provide fast access and a simple memory layout.
    • Dynamic arrays offer flexibility but require occasional copying during resizing.
    • Choosing between static arrays, dynamic arrays, and linked lists depends on the need for speed, memory efficiency, or flexibility.
