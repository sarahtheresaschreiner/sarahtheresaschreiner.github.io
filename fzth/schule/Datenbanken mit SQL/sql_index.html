    INNER JOIN - Tabellen verbinden

    Wenn man join schreibt meint man INNER JOIN, es ist der default

    Was wir wollen (implizite Syntax):

    SELECT
    Tab1.Name,
    Tab2.Name,
    Tab1.Id,
    Tab2.Id
FROM
    Tab1, Tab2
WHERE
    Tab1.Id = Tab2.Id;

    Wie es geht (explizite Syntax):

    SELECT
    Tab1.Name,
    Tab2.Name,
    Tab1.Id,
    Tab2.Id
FROM
    Tab1 JOIN Tab2 ON Tab1.Id = Tab2.Id;

    INNER JOIN: nur wenn für beide Tabellen ein Wert existiert

    Mehr als zwei Tabellen kombinieren (geht genauso)

    SELECT Kunde.Nachname
FROM
  Mitarbeiter
  JOIN betreut ON betreut.MitarbeiterID = Mitarbeiter.MitarbeiterID
  JOIN Kunde ON betreut.KundenID = Kunde.KundenID
  JOIN Fahrzeug ON Fahrzeug.KFZ = Mitarbeiter.KFZ

    LEFT JOIN

    Ein normaler JOIN (oder INNER JOIN) gibt nur Zeilen zurück, die in beiden Tabellen eine Entsprechung haben. Ein LEFT JOIN hingegen gibt alle Zeilen der linken Tabelle zurück – auch wenn es in der rechten Tabelle keine passenden Daten gibt. In diesem Fall werden die fehlenden Spalten mit NULL aufgefüllt.

    SELECT Mitarbeiter.Nachname, Fahrzeug.Bezeichnung 
FROM Mitarbeiter 
    LEFT JOIN Fahrzeug ON Mitarbeiter.KFZ = Fahrzeug.KFZ;

    Achtung, wenn man auf Werte zugreift immer über die Seite die alle Werte (h: Mitarbeiter) hat, das ist immer die Linke

    Keywords

    • Distinct : jedes Element nur einmal -> Unique
    • LIMIT x nur x Spalten anzeigen --> sinnvoll bei großen Datenbanken
    • company.spalte as spaltenbeschriftung anderen Head zuweisen
    • WHERE-Teil
      • tab.ID in abf alle die in der abfrage sind; auch möglich {2,3,4}
      • tab.ID not in abf alle die nicht in sind

    Aggregatsfunktionen

    • COUNT(*) – Zählt die Anzahl der Zeilen (Datensätze).
    • SUM(Spalte) – Berechnet die Summe aller Werte in einer Spalte.
    • AVG(Spalte) – Berechnet den Durchschnittswert einer Spalte.
    • MIN(Spalte) – Findet den kleinsten Wert in einer Spalte.
    • MAX(Spalte) – Findet den größten Wert in einer Spalte.
    • ROUND(Wert, Stellen) - Rundet auf die Anzahl der Stellen nach dem Komma

    Agg-Funcs können nicht bei WHERE verwendet werden -> sub select machen

    GROUP BY - Gruppieren

    SELECT 
    *
FROM 
    Tabelle
GROUP BY 
    Tabelle.Spalte
    • erstellt für jedes unique value eine Gruppe
    • Agg.-func. werden auf jede Gruppen einzeln angewendet

    HAVING: Gruppen filtern

    Während WHERE einzelne Zeilen filtert, bevor gruppiert wird, filtert HAVING die fertigen Gruppen.

    Beispiel: Zeige nur Orte, in denen das Durchschnittsgehalt über 2200€ liegt:

    SELECT Ort, AVG(Gehalt) FROM Mitarbeiter GROUP BY Ort HAVING AVG(Gehalt) > 2200;

    ORDER BY

    • ORDER BY Tabelle.Spalte DESC
    • Sortiert nach dem angegebenen Wert (absteigend: DESC, aufsteigend ASC)

    Insert

    Eine Zeile einfügen

    INSERT INTO Mitarbeiter (Vorname, Nachname, Gehalt, AbteilungsID) 
VALUES ('Anna', 'Schmidt', 3200, 3);
    • INSERT INTO : Tabelle, welche Felder bekommen Inhalt
    • VALUES : die Inhalte die eingefügt werden

    Update

    Werte ändern

    UPDATE Mitarbeiter 
SET Gehalt = 3500 
WHERE Nachname = 'Schmidt';
    • UPDATE : welche Tabelle
    • SET : was
    • WHERE: Bedingung, ansonsten alle

    Delete

    Zeile löschen

    DELETE FROM Mitarbeiter 
WHERE Nachname = 'Schmidt';
    • DELETE FROM : aus welcher Tabelle soll gelöscht werden
    • WHERE : bei welcher Bedingung soll gelöscht werden
