    Encoding Categorical Variables

    https://www.geeksforgeeks.org/machine-learning/categorical-data-encoding-techniques-in-machine-learning/

    Types of categorical data

    For ML you need numeric input most of the time. Thatway we have to encode categorical input (e.g. 'blue', 'red'). There are two types of categorical data:

    Nominal Data

    • categories without any inherent order or ranking
    • simple labels, e.g. color
    • Encoding options: One-Hot Encoding, Label Encoding

    Ordinal Data

    • defined order or ranking, that is important
    • e.g. 'Low', 'Medium', 'High'
    • Encoding: Ordinal Encoding

    Techniques

    Label encoding

    Assigning each variable a unique integer

    • Used in tree-based models like Decision Tree or XGBoost
    • Pro: Simple and memory-efficient
    • Cons: implicit order, could be misinterpreted by non-tree models.

    Implemented with LabelEncoder from sklearn

    from sklearn.preprocessing import LabelEncoder as le

data = ['Red', 'Green', 'Blue', 'Red']
encoded_data = le.fit_transform(data)

    Output: Encoded Data: [0,1,2,0]

    One-Hot Encoding

    Converts the categories into binary columns, each column representing a category. 1: presence, 2: absence

    • Linear models, logistic regression, neural networks
    • Pros: Does not assume order
    • Cons: Can cause high dimension and sparse data when feature has many categories

    Implementing with Pandas get_dummies()

    import pandas as pd

data = ['Red', 'Green', 'Blue', 'Red']
df = pd.DataFrame(data, columns=['Color'])
one_hot_encoded = pd.get_dummies(df['Color'])

    Ordinal Encoding

    Mapping categories to integers, while preserving their natural order

    • Order features like ratings or education levels
    • Pros: Maintains order; reduces dimensionality.
    • Cons: Not suitable for nominal categories.

    Implementing with Sklearn NominalEncoder

    import sklearn.preprocessing import OrdinalEncoder
data = [['Low'], ['Medium'], ['High'], ['Medium'], ['Low']]

oe = OrdinalEncoder(categories=[['Low','Medium','High']])
encoded_data = oe.fit_transform(data)

    Output: encoded_data: [[0.][1.][2.][1.][0.]]
