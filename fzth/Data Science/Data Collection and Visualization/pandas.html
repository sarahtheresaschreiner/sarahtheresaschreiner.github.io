    Core Concepts

    DataFrame

    • Tabel
    • Row and Column
    • All datatypes (int, string, etc.)
    • Constructor: pd.DataFrame({'Col1': [1,2], 'Col2': [3,4]})
    • Row labels: 0,1,...,#rows
    • Clarifying row labels: pd.DataFrame({'Col1': [1,2], 'Col2': [3,4]}, index=['Row1', 'Row2'])

    Series

    • List
    • Fancy Dict
    • Thinking of a single-column-dataFrame
    • Constructor: pd.Series([1,2], index=['Row1', 'Row2'], name='mSeries')
    • index: row labels, optional (as in DataFrame)
    • name: name of the Series, optional

    CSV ("Comma Seperated Values")

    • File with values that are normally seperated by commas

    Syntax

    • CSV: pd.read_csv(path)
    • df.to_csv(path): save DataFrame df as a csv file
    • index_col: which col of the csv should be index
    • Attributes
    • shape: (#rows,#cols)
    • col1: series of the values from col1
    • Methods
    • head: first five rows
    • set_index(row_name): makes the row the new index
    • df.col == condition: returns a bool-series with True/False for each row
      • You can use it in loc to get only the rows that match your condition, you can also combine conditions bool1 & bool2 or use pipe | for or
    • col.isin(list_of_accepted_vals): select data whose values are in the list
    • col.describe(): sumarises the values, e.g. for ints mean etc.
    • col.mean(): mean
    • col.unique(): all unique values (all that are there, but not its repeating)
    • col.value_counts(): how often each value occurs, returns a Series with inde being the value and amount as value
    • Map and Apply
    • row.idxmax: returns the index of the maxvalue
    • replace('old', 'new'): replaces all matching values
    • Accessing
    • A Series/Col: df[col1] oder df.col1
    • A field/value: df[col1][row]
    • iloc and loc both use row_first, col_second in pandas
      • Index-based: df.iloc[row,col] -> possible to only use row
      • Index Rules
      • Label-based: `df.loc[row,col]
      • both from/to are inclusiv for loc
      • can do both index and label
      • reviews.loc[:, ['taster_name', 'taster_twitter_handle', 'points']]
    • Assigning
    • df[col] = 'hello world': constant value for all rows
    • reviews['index_backwards'] = range(len(reviews), 0, -1): different value for each row
    • inplace=True
    • add this if you want to change the df and not just want to get a changed copy
    • you can only add this on dataFrames, that don't return a completely new dataFrame
    • you can also just do: df = df.head()

    Index Rules

    • all: [:]
    • all up to (exclusiv): [:3]
    • from (inclusiv) to (exclusive): [1:3]
    • list of values[[1,2,3], col]
    • last 5: [-5:]

    Map and Apply

    • row.map()
    • reviews.points.map(lambda p: p - review_points_mean)
    • Map returns a Series of transformed values, transformed by your given function (that ideally takes one value and returns one value)
    • df.apply()
    • reviews.apply(remean_points, axis='columns')
    • same but on a complete row
    • axis='index'would do each column
    • You can also use operaters, which is faster and easier to read:
    • reviews.country + " - " + reviews.region_1
    • reviews.points - review_points_mean

    Grouping

    • groupby()
    • creates a group for each unique value of the col
    • e.g.: reviews.groupby('points').points.count(), it does the same as value_counts()
    • you can use apply() on groups, it applys on every group as a df
    • agg([len, min, max]): returns a df with a col for each of these operations for every group

    Multi-indexes

    • can occure when grouping
    • require two levels of labels to retrieve a value
    • reset_index() converts your df back to single-index, the old multi-indexes become now the first cols

    Sorting

    • sort_values(by='len'): ascending (lowest first)
    • ascending=False: descending order
    • by=['firstArg', 'secArg'] sort by more than one value
    • sort_index(): sort by index

    Dtypes

    • reviews.price.dtype: you can also call it on the df to get a series with dtype for each col
    • it tells howmany bits it uses e.g. on int (int64) or float
    • Strings are object-type
    • astype('float64'): converting to float dtype
    • index has a dtype aswell
    • NaN: empty fields, dtype=float64
    • row.isnull() | row.notnull(): get all, that are (not) empty (NaN)
    • fillna(): replaces all NaNs with the given input

    Renaming

    • reviews.rename(columns={'points': 'score'})
    • reviews.rename(index={0: 'firstEntry', 1: 'secondEntry'})
    • supports a variety of input formats, usually dict
    • reviews.rename_axis("wines", axis='rows').rename_axis("fields", axis='columns')

    Combining

    • concat([df1,df2])
    • e.g. two dfs with same cols can be added together
    • join()
    • left.join(right, lsuffix='_CAN', rsuffix='_UK')
    • if in right is a matching index it will be added to left, the col names get a suffix
    • pd.merge(left,right, on='user_id')
    • join two dfs on a declared column, that is the same in both dataFrames.
