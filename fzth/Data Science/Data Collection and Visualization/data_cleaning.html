    Handling missing values

    1. How many missing values are there: missing_values_count = df.isnull().sum()
      • maybe look at the total percentage for the hole df
    2. Why does it miss
      • Is this value missing because it wasn't recorded or because it doesn't exist?
      • if it wasnt recorded you can try to guess it - "Imputation"
    3. Drop missing values
      • dropna() to drop all rows with a missing value
      • dropna(axis=1) to drop all coloumns with a missing value
      • Look at how many are left compared to before dropping
    4. Fill in missing data
      • fillna(0) to replace every NaN with a '0'
      • .fillna(method='bfill', axis=0).fillna(0) to replace all NaN's with the value underneath it. The remaining NaN's become '0'

    Scaling and Normalization

    • in scaling, you're changing the range of your data, while

    • in normalization, you're changing the shape of the distribution of your data.

    • Minmax-scaling: scaled_data = minmax_scaling(original_data, columns=[0]), with min = 0 and max = 1

    • Boxcox-Normalizing (Normal Distribution/Bell Curve): normalized_data = stats.boxcox(original_data)
    • from scipy import stats

    Parsing Dates

    import datetime

    1. check if the columns dtype is actually datetime not object
    2. Convert to datetime:
      • landslides['date_parsed'] = pd.to_datetime(landslides['date'], format="%m/%d/%y")
      • format is current format, before parsing
      • let pd guess the format (e.g. if you have different formats): landslides['date_parsed'] = pd.to_datetime(landslides['Date'], infer_datetime_format=True)
    3. Get the data: day_of_month_landslides = landslides['date_parsed'].dt.day

    Using other encoding methods

    • python is allways utf-8. That means that if you encode with e.g. ASCII, python will try but cant't work out all characters. It can replace these values with the unknown character you see, when you decode.
