    Decision Trees

    • The leaf is where the prediction is made

    Interpret your data

    With df.describe() you get a DataFrame with a column for each of your columns. The rows:

    • count: how many not NaN values in that col
    • mean: average value
    • std: Standard deviation
    • min: min value
    • max: max value
    • 25%: a quarter of the values is smaller/equal (first quartile)
    • 50%: the median
    • 75%: 3 quarters are smaller/equal (third quartile)

    Choose Target and features

    • Selecting the prediciton target y
      • y = melbourne_data.Price
    • Features are the values that are the input for your model, based on which the prediction is made. They are called X
      • melbourne_features = ['Rooms', 'Bathroom', 'Landsize', 'Lattitude', 'Longtitude']
      • X = melbourne_data[melbourne_features]
    • Now you've got the DataFrame X and the Series y

    Building the Model

    from sklearn.tree import DecisionTreeRegressor

    The steps to building and using a model are:

    1. Define: What type of model will it be? A decision tree? Some other type of model? Some other parameters of the model type are specified too. Specify a number for random_state to get the same results each run. > melbourne_model = DecisionTreeRegressor(random_state=1)
    2. Fit: Capture patterns from provided data. This is the heart of modeling. > melbourne_model.fit(X, y)
    3. Predict: Just what it sounds like > melbourne_model.predict(X.head())
    4. Evaluate: Determine how accurate the model's predictions are.

    Model Validation

    • Summaries the results into a metric:
      • Mean Absolute Error (MAE)
      • Error: error=actual-predicted
      • Absolut: all positiv values
      • Mean: average
      • On average our predictions are off by about X
      • from sklearn.metrics import mean_absolute_error
      • mean_absolute_error(y, predicted_home_prices)
    • "In-Sample" : Using the same data for training and testing
      • don't use all the data for fitting but save some for validating the prediction
    • Split train and test data (Out-of-sample)
      • from sklearn.model_selection import train_test_split
      • train_X, val_X, train_y, val_y = train_test_split(X, y, random_state = 0)
      • mean_absolute_error(val_y, val_predictions)

    Overfitting and Underfitting

    • When your model is to fitted to your training-data, it wouldn't work with other test data
    • This happens e.g. if a decision tree has a too high depth
    • Underfitting is the opposite, here the tree has two less decisions and will work poorly on training and test data

    Implementing

    from sklearn.metrics import mean_absolute_error
from sklearn.tree import DecisionTreeRegressor

def get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y):
    model = DecisionTreeRegressor(max_leaf_nodes=max_leaf_nodes, random_state=0)
    model.fit(train_X, train_y)
    preds_val = model.predict(val_X)
    mae = mean_absolute_error(val_y, preds_val)
    return(mae)

# compare MAE with differing values of max_leaf_nodes
for max_leaf_nodes in [5, 50, 500, 5000]:
    my_mae = get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y)
    print("Max leaf nodes: %d  \t\t Mean Absolute Error:  %d" %(max_leaf_nodes, my_mae))
